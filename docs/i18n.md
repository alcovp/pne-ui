i18n-гид для pne-ui
===================

Зачем
-----
Компоненты не принимают тексты через пропсы и полагаются на `react-i18next`. Файл описывает, какие ключи должны быть зарегистрированы у хоста и откуда их брать.

Минимальная инициализация
-------------------------
```ts
import i18n from 'i18next'
import {initReactI18next} from 'react-i18next'

i18n.use(initReactI18next).init({
    lng: 'ru',
    fallbackLng: 'en',
    defaultNS: 'translation',
    resources: {
        ru: {translation: {/* см. ключи ниже */}},
    },
})
```
Достаточно один раз импортировать конфиг в корне приложения или Storybook, чтобы `pne-ui` подтянулся к уже созданному контексту.

Базовые ключи (без префиксов)
-----------------------------
`search`, `cancel`, `save`, `create`, `clear`, `clear.all`, `react.chooseOne`, `in`, `MID`

Блоки ключей
------------
- **SearchUI/панель фильтров:** `react.searchUI.filters`, `react.searchUI.addCriterion`, `react.searchUI.selected`, `react.searchUI.available`, `react.searchUI.include`, `react.searchUI.exclude`, `react.searchUI.all`, `react.searchUI.none`, `react.searchUI.allExcluding`, `react.searchUI.onlyEnabledStatus`, `react.searchUI.addSessionStatusTitle`, `react.searchUI.transactionSessionStatusGroup`, `react.searchUI.countrySelectPlaceholder`, `react.searchUI.poorSearchEngine`.
- **Шаблоны поиска:** `react.searchUI.template`, `react.searchUI.template.create`, `react.searchUI.template.update`, `react.searchUI.template.name`, `react.searchUI.template.name.confirmRewrite`, `react.searchUI.template.newModal.title`.
- **Расширенный поиск:** `advancedSearch.addCriteria`, `advancedSearch.noRows`.
- **Перформанс-репорт и группировки:** `performanceReport.grouping`, `performanceReport.convertAllTo`, `performanceReport.userCurrency`.
- **Абстрактные селекторы:** `abstractSelector.available`, `abstractSelector.added`, `abstractSelector.select.all`.

Префиксы для enum/списков
-------------------------
- `react.CriterionTypeEnum.*` — значения из `CriterionTypeEnum` (`EXACT`, `ORDERS_SEARCH`, `CURRENCY`, `THREE_D`, `STATUS`, `MERCHANT`, `ENDPOINT`, `RESELLER`, `PROCESSOR`, `MANAGER`, `PROJECT`, `COMPANY`, `GATE`, `DEALER`, `DATE_RANGE`, `DATE_RANGE_ORDERS`, `PROJECT_CURRENCY`, `CARD_TYPES`, `COUNTRIES`, `GROUPING`, `TRANSACTION_TYPES`, `TRANSACTION_STATUS`, `RECURRENCE_TYPE`, `RECURRENCE_STATUS`, `MFO_CONFIGURATION_TYPE`, `MARKER_TYPE`, `MARKER_STATUS`, `PROCESSOR_LOG_ENTRY_TYPE`, `ERROR_CODE`, `TRANSACTION_SESSION_STATUS`).
- `react.ExactCriterionSearchLabelEnum.*` — значения из `ExactCriterionSearchLabelEnum` (`ALL`, `NAME`, `DESCRIPTION`, `TAGS`, `IDENTIFIER`, `BEAN`, `END_POINT_GROUP_ID`, `ID`, `AMOUNT`, `FINAL_CLEARING_DATE`, `MANAGER`, `SERIAL_NUMBER`, `INVOICE`, `CARD_FROM_RECURRENCE_NUMBER`, `FIRST_6`, `LAST_4`, `FIRST_6_LAST_4`, `ORDER_IDENTIFIER`, `EMAIL`, `LOGIN`, `PRINCIPAL_DEALER`, `PRINCIPAL_MANAGER`, `PRINCIPAL_MERCHANT`, `PRINCIPAL_RESELLER`, `PRINCIPAL_SUPERIOR`, `END_POINT_IDENTIFIER`, `END_POINT_GROUP_IDENTIFIER`).
- `react.DateRangeSpecType.*` — `EXACTLY`, `TODAY`, `YESTERDAY`, `THIS_WEEK`, `LAST_WEEK`, `THIS_MONTH`, `LAST_MONTH`, `DAYS_BEFORE`, `HOURS_BEFORE`, `DATE_INDEPENDENT`.
- `orders.orderlist.search.datesType.*` — `SESSION_CREATED`, `SESSION_STATUS_CHANGED`, `TX_CREATED`, `BANK`, `TX_SETTLED`, `TX_UNSETTLED`.
- `searchLabel.*` — значения из `ORDER_SEARCH_LABELS` (см. `src/component/search-ui/filters/types.ts`, например `merchant_invoice_id`, `order_id`, `processor_order_id`, …). При необходимости маппируйте на собственные человеко-читаемые лейблы.
- `performanceReport.groupType.*` — значения из `GroupingType` (`MERCHANT`, `MANAGER`, `PROJECT`, `CURRENCY`, `ENDPOINT`, `CARD_TYPE`, `GATE`, `PROCESSOR`, `MID`, `COUNTERPARTY`, `PROJECT_CODE`, `DATE`, `MONTH`, `DAY`, `CLOSE_DAY`, `SETTLEMENT_DAY`, `SETTLEMENT_MONTH`).
- `react.StatusCriterionEnum.*` — `ANY`, `DISABLED`, `ENABLED`.
- `react.MarkerStatusCriterionEnum.*` — `any`, `unprocessed`, `processed`.
- `react.ThreeDCriterionEnum.*` — `ANY`, `NO`, `YES`.

Шаблон ресурсов
---------------
```ts
export const resources = {
  ru: {
    translation: {
      search: 'Поиск',
      cancel: 'Отменить',
      clear: 'Очистить',
      'react.searchUI.filters': 'Фильтры',
      'react.searchUI.template': 'Шаблон',
      'react.CriterionTypeEnum.EXACT': 'Точный поиск',
      // ...остальные ключи и enum значения...
    }
  }
}
```
Если библиотека не найдёт ключ, i18next вернёт сам идентификатор — используйте это как индикатор пропущенных строк.
